Inugami-NG : inu-code

Simplify Adding Code Examples to Your Angular Applications

Many documentation-driven applications require the inclusion of code examples. While various libraries handle syntax highlighting effectively, the challenges of writing and maintaining these examples remain unresolved.

The inu-code component takes a different approach. Instead of embedding examples directly into your pages, it allows you to store them in a separate XML file. This method offers several advantages:

Easier Maintenance
A single file contains all your examples. Since they are stored in XML format, they can even be easily automated.
Reduced Application Bundle Size
Hardcoding resources directly into an Angular application increases its overall size. By moving these to external files, you mechanically reduce the application's weight, leading to faster loading times.

The example retrieval mechanism caches all fetched elements. If you use the same file for multiple examples, only a single HTTP request will be performed.

Inugami-NG : inu-open-api

Streamline your OpenAPI contract visualization

Many projects need to bridge the communication gap between teams developing and consuming REST services. While the standard Spring Boot approach can be used to display a service contract, it only reflects the current state of the application's web services.

When managing API versioning, it becomes essential to communicate all available API versions. These versions may reside on different instances (or even separate clusters), allowing for precise control over backward compatibility.

In such scenarios, there are few solutions available for centrally displaying different API versions. The inu-open-api component is designed to help your documentation platform display various service contracts in a centralized way.

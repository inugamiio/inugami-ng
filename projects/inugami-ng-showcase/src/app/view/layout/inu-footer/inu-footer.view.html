Inugami-NG : inu-footer

The inu-footer component is designed to display information related to your application, such as:

  • The application name
  • Its version
  • And its execution environment

This component is featured in the footer of this showcase.

Usage

This component is designed to be defined directly in the main app.html file.

As with any standalone component, it must be imported directly into the component that uses it.

Application information is provided through the provideCacheTracking provider, defined in the app.config.ts file.

We use Angular Signals to manage application metadata (name, environment, version, and session UID). These signals are then passed to the provideCacheTracking function, which registers them as globally accessible providers.

It is also possible to add information inside the component.

Inugami-NG : Icons

Bring Your Applications to Life with Icons

Icons are an essential part of any application. The human brain is designed to identify shapes and colors; a simple icon is far more recognizable than a long block of text.

Inugami provides a comprehensive icon set. Following the core Inugami philosophy, this system can be extended while maintaining a consistent workflow.

When using icons dynamically, the requested icon might not exist. In such cases, you can define a default icon.

Dependency

While inugami-icons comes with a built-in set, you have full control over which icons are available in your application. You can use the default set, modify it, or extend it with your own custom icons.

To enable icons, you must register the desired set in your main configuration file (usually app.config.ts):

The InugamiIconsUtils utility class is not only used to register an icon set but also serves as a central registry to retrieve a specific icon or list all icons currently defined in your application.

When registering icons, the InugamiIconsUtils utility class expects to receive a list of IconDefinition objects. You can override an icon by redefining it with the same name.

Best Practices for Icon Design (Inkscape)

To ensure your icons are processed correctly by the Inugami build script, please follow these guidelines when working in Inkscape.

  1. Document Setup
    Grid System
    Use a fixed-size document (e.g., 24x24 px).
    ViewBox
    Ensure your main drawing area matches the viewBox defined in the build script (default is 0 0 24 24).
    Units
    Always use pixels (px) as the primary unit to avoid scaling issues.
  2. Layer Organization
    Naming
    The script uses the Layer Label (inkscape:label) as the icon's unique ID.
    Layer as Icon
    Each icon must be isolated in its own top-level layer.
    Exclusion
    To prevent a layer from being exported, prefix its name with an underscore (e.g., _background or _draft).
  3. Paths and Geometry
    Convert to Path
    Ensure all shapes (circles, rectangles, text) are converted to paths (Ctrl+Shift+C).
    Simplicity
    Use Path > Union to combine overlapping shapes into a single path when possible. This reduces file size and improves rendering performance.
    Grouping
    Avoid unnecessary nested groups within a layer unless strictly required for multi-color icons.
  4. Handling Colors
    Monochrome Icons
    For standard UI icons, use a solid black fill. The script will allow the application to override the color via CSS.
    Multi-color Icons (Brands/Flags)
    You can use specific colors. The script is configured to preserve styles for complex assets like the Inugami logo or national flags.
    Cleaning
    Avoid using "Filters" or "Blurs" as they often rely on complex SVG definitions that may not render consistently across all browsers.
  5. Alignment
    Centering
    Always center your artwork within the 24x24 frame.
    Safe Zone

    To maintain a perfect visual balance across the entire set, we recommend setting up Guides (marge/safe zone) within Inkscape

    • Safe Zone (22x22px) : Place guides at 1px from each edge. Your main icon shapes should stay within this area to avoid being cut off or appearing too close to other UI elements.
    • Action Zone (20x20px)
    • For denser icons, try to keep the "heavy" part of the design within 2px margins.
    • Optical Centering: Use a horizontal and vertical guide at 12px to ensure your icon is optically centered, which is sometimes different from geometric centering.

In Inkscape, you can create these guides once and lock them, or save your 24x24 setup as a Template for all future Inugami icons.

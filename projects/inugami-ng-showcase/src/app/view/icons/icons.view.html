Inugami-NG : Icons

Bring Your Applications to Life with Icons

Icons are an essential part of any application. The human brain is designed to identify shapes and colors; a simple icon is far more recognizable than a long block of text.

Inugami provides a comprehensive icon set. Following the core Inugami philosophy, this system can be extended while maintaining a consistent workflow.

Dependency

While inugami-icons comes with a built-in set, you have full control over which icons are available in your application. You can use the default set, modify it, or extend it with your own custom icons.

To enable icons, you must register the desired set in your main configuration file (usually app.config.ts):

The InugamiIconsUtils utility class is not only used to register an icon set but also serves as a central registry to retrieve a specific icon or list all icons currently defined in your application.

When registering icons, the InugamiIconsUtils utility class expects to receive a list of IconDefinition objects. You can override an icon by redefining it with the same name.

Inugami-NG : InuErrorServices

Usage

Exception handling is a critical aspect of any application. It is important to notify users whenever the server encounters issues.

Inugami-ng's error handling is built upon the core Inugami error management system. This logic relies on unique error codes to track even the slightest anomalies.

To learn more about Inugami error management: Inugami Exception management

Usage is straightforward via the forkJoin method, which wraps the standard RxJS implementation. This method proactively handles potential error cases and automatically displays a toast notification when an error occurs. It returns an Observable, allowing you to retrieve either the successful response data or a ProblemDTO object.

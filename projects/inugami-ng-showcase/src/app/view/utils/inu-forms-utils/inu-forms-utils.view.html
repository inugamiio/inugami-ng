Inugami-NG : InuFormsUtils

This utility class is a great addition to the Inugami-NG ecosystem. It bridges the gap between Angular Signals and Observables while providing a clean way to handle form validation states.

onChanged

Converts a Signal-based model into an Observable stream to track value changes with built-in performance optimizations.

Ideal for triggering side effects (like API calls or logging) when a model changes.

Debounce
Defaults to 50ms to prevent over-triggering during rapid input.
Filtering
Only emits when the value actually changes (distinctUntilChanged).
Memory Management
Automatically cleans up the subscription when the calling context is destroyed (takeUntilDestroyed).

isValid

Performs a comprehensive validation check on a FieldTree structure.

Quickly assess the health of a form and retrieve specific metadata for any failing fields.

It iterates through the form keys, inspects the validationState of each signal-based field, and compiles a list of failures including their nodeState (dirty, touched, etc.).
